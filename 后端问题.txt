❯ 重新构建镜像耗费流量吗？

● 看一下 Dockerfile 了解构建方式：

● Read 1 file (ctrl+o to expand)

● 结论：只改 Python 代码（不改依赖）时，重建基本不耗流量。

  Dockerfile 分层结构决定了缓存命中情况：

  ┌─────────────────────────┬──────────────┬──────────────────────────────────────────────────────────┐
  │           层            │     内容     │                        是否走网络                        │
  ├─────────────────────────┼──────────────┼──────────────────────────────────────────────────────────┤
  │ FROM python:3.10        │ 基础镜像     │ 本地有缓存，不下载                                       │
  ├─────────────────────────┼──────────────┼──────────────────────────────────────────────────────────┤
  │ apt-get 字体安装        │ 系统包       │ 本地有缓存，不下载                                       │
  ├─────────────────────────┼──────────────┼──────────────────────────────────────────────────────────┤
  │ uv sync --frozen 依赖层 │ Python 包    │ 只要 pyproject.toml / uv.lock 没变，全部命中缓存，不下载 │
  ├─────────────────────────┼──────────────┼──────────────────────────────────────────────────────────┤
  │ COPY ./app 代码层       │ 你的业务代码 │ 只是本地文件复制，不走网络                               │
  └─────────────────────────┴──────────────┴──────────────────────────────────────────────────────────┘

  所以日常改 .py 文件后重建，流量消耗 ≈ 0。

  只有以下情况才会重新下载：
  - 修改了 pyproject.toml 或 uv.lock（新增/升级依赖）
  - 本地 Docker 层缓存被手动清除（docker builder prune）
  - 换了新机器，本地没有缓存

  当前情况可以放心用：
  docker compose up -d --build backend
