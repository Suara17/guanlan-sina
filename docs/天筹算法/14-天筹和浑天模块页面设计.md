基于您上传的计划书《天工·弈控 ——面向离散制造的“视-空协同”智适应操作系统》，以及前序沟通中确定的 React 19 + Tailwind + 深色科技风技术栈，为您规划“天筹”与“浑天”两个核心模块的 Web 页面设计方案。

这两个模块分别对应系统架构中的 **决策层（Decision）** 和 **验证层（Validation）**。

# 核心模块设计方案：天筹 & 浑天

## 1. 决策优化模块 —— “天筹” (Tianchou)

**定位：** 系统的“最强大脑”。基于运筹学和启发式算法，解决生产排程拥堵、AGV 路径冲突等多目标优化问题。

**路由：** `/app/optimization`

### 1.1 页面布局与功能分区

页面采用 **左右分栏** 结构，强调“问题输入”与“方案对比”的直观性。

* **左侧：参数与约束配置区 (25%)**
* **标题：** 优化目标设定 (Optimization Goals)。
* **控件组：**
* **目标权重滑块：** 允许用户在“效率优先”、“能耗最低”、“设备均衡”之间拖拽平衡。例如，将“时间成本”权重拉到 80%，“设备损耗”权重降至 20%。
* **约束条件开关：** “避开维护中设备”、“强制夜间停机”、“AGV 充电优先”。


* **触发按钮：** 一个显著的 **“生成优化方案” (Generate Strategy)** 按钮，点击后触发加载动画（模拟算法计算过程）。


* **右侧：方案推演与对比区 (75%)**
* **核心视图：双轨制对比面板 (Dual-Track Comparison)**
* **视图 A (当前/基准)：** 展示当前的生产排程甘特图 (Gantt Chart) 或 物流热力图，标注出红色的“拥堵点”或“闲置空窗期”。
* **视图 B (天筹优化后)：** 展示优化后的甘特图，用绿色高亮显示“消除的等待时间”或“缩短的路径”。


* **KPI 预测卡片：**
* 预计总耗时：`4h 15m` ➔ `3h 20m` (📉 21%)
* 设备平均负载率：`65%` ➔ `82%` (📈 17%)


* **决策建议：** 底部显示“司南”生成的自然语言解释，例如：“建议调整 #3 机械臂的任务优先级，可消除瓶颈。”



### 1.2 关键交互设计

* **交互逻辑：** 用户调整左侧的“权重滑块”时，右侧的预测数据实时（或防抖后）变化，体现“智适应”特性。
* **视觉隐喻：** 使用流动的线条连接左侧参数和右侧结果，暗示数据流向。

### 1.3 技术实现建议

* **图表库：** 使用 `Recharts` 的 `BarChart` (Range Bar) 模拟简易甘特图，或使用专门的甘特图组件。
* **布局：** Tailwind Grid 布局。

## 2. 验证仿真模块 —— “浑天” (Huntian)

**定位：** 系统的“数字孪生沙盘”。在虚拟环境中进行高强度压力测试，验证方案的鲁棒性，确保物理世界执行零风险。

**路由：** `/app/simulation`

### 2.1 页面布局与功能分区

页面采用 **沉浸式大屏** 结构，模拟类似游戏或工业软件的控制台。

* **中央：数字孪生主视口 (Main Viewport)**
* **内容：** 展示车间 2.5D 平面图或简易 3D 模型。
* **动态元素：**
* 代表工单的“光点”在工位间流动。
* 代表 AGV 的“图标”在路径上移动。
* 故障模拟：随机在某个设备上出现“🔥”或“⚠️”图标，观察系统能否自动绕开。




* **底部：时间轴控制台 (Time Control Bar)**
* **播放控制：** 播放/暂停，**倍速播放** (1x, 10x, 50x)。用于演示“一天的工作量在一分钟内跑完”。
* **进度条：** 拖拽回溯，查看某个拥堵时刻的现场快照。


* **右侧：压力测试面板 (Stress Test Panel)**
* **场景预设：** 下拉选择测试剧本，例如“物料延迟 2小时”、“3号机突发停机”、“订单量激增 200%”。
* **实时日志 (Console)：** 滚动显示模拟日志。
* `[09:15:02] 模拟注入：#2 AGV 电量耗尽`
* `[09:15:05] 系统响应：任务自动重新分配给 #4 AGV`
* `[09:15:08] 验证结果：生产节拍未受影响 ✅`





### 2.2 关键交互设计

* **“极端工况”演示：** 专门设计一个 **“注入故障”** 按钮。用户点击后，主视口中的一台设备变红，系统立即在该视口中通过动画展示路径重规划的过程（例如，AGV 绕路）。
* **结果反馈：** 仿真结束后，弹出一份“鲁棒性评分报告”，给出 0-100 的分数。

### 2.3 技术实现建议

* **渲染引擎：** 鉴于 Web 演示的轻量化需求，建议使用 `Canvas` API 或 SVG 动画实现 2.5D 效果，而非重型的 Three.js（除非有现成模型）。
* **动画：** 使用 CSS Keyframes 或 `Framer Motion` 处理 AGV 的平滑移动。

## 3. 页面间的数据流转（串联演示剧本）

为了演示的连贯性，建议按以下剧本串联各页面：

1. **Dashboard (首页)：** 发现“产能不足”的报警。
2. **天筹 (优化页)：** 用户进入，调整参数，生成一个新的“提速方案”。
3. **浑天 (仿真页)：** 在下发到真实产线前，先在浑天中跑一遍。用户点击“10倍速验证”，看到模拟结果一切正常（鲁棒性 98%）。
4. **最终操作：** 验证通过，点击“部署到产线”，闭环回到 Dashboard，看到指标回升。

这个设计方案严格遵循了计划书中“双轨制优化”和“压力测试”的核心概念，同时适配了当前 Web Demo 的开发能力。
