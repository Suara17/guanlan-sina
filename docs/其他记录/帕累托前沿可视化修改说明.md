# 帕累托前沿可视化功能修改说明

## 概述

本次修改实现了将后端算法生成的帕累托前沿数据传递到前端显示的功能，替换了原有的模拟数据，使前端能够展示真实的优化结果。

## 问题背景

原有实现存在以下问题：
1. 前端 `ParetoTriplot.tsx` 组件使用模拟数据生成图表
2. 后端算法生成的所有解数据没有传递到前端
3. 数据库没有存储所有解数据

## 修改内容

### 1. 后端修改

#### 1.1 数据库模型 (`backend/app/models.py`)

在 `OptimizationTask` 模型中添加 `all_solutions` 字段：

```python
# 存储所有解数据（用于前端帕累托前沿可视化）
all_solutions: dict = Field(default={}, sa_column=Column(JSONB))
```

#### 1.2 API 路由 (`backend/app/api/routes/tianchou.py`)

**修改点 1**：在后台任务执行时保存所有解数据

```python
# 保存所有解数据（用于前端帕累托前沿可视化）
all_solutions = results.get("all_solutions", [])
all_solutions_data = [
    {
        "f1": float(sol["f1"]),
        "f2": float(sol["f2"]),
        "f3": float(sol.get("f3")) if sol.get("f3") is not None else None,
    }
    for sol in all_solutions
]
task.all_solutions = {"solutions": all_solutions_data}
db.commit()
```

**修改点 2**：新增 API 端点获取所有解数据

```python
@router.get("/tasks/{task_id}/all-solutions")
async def get_all_solutions(task_id: str, session: SessionDep) -> Any:
    """
    获取所有解数据（用于帕累托前沿可视化）
    """
    task = session.get(OptimizationTask, uuid.UUID(task_id))
    if not task:
        raise HTTPException(status_code=404, detail="任务不存在")

    solutions = task.all_solutions.get("solutions", [])

    return {
        "task_id": task_id,
        "industry_type": task.industry_type,
        "solutions": solutions,
        "total_count": len(solutions),
    }
```

### 2. 前端修改

#### 2.1 类型定义 (`frontend/pages/Tianchou/types/tianchou.ts`)

新增类型：

#### 2.2 服务层 (`frontend/pages/Tianchou/services/tianchouService.ts`)

新增获取所有解的方法：

```typescript
async getAllSolutions(taskId: string): Promise<AllSolutionsResponse> {
  const response = await fetch(`${API_BASE}/tasks/${taskId}/all-solutions`)
  if (!response.ok) {
    throw new Error(`Failed to get all solutions: ${response.statusText}`)
  }
  return response.json()
}
```

#### 2.3 组件重写 (`frontend/pages/Tianchou/components/ParetoTriplot.tsx`)

**主要改进**：

1. **使用真实数据**：通过 API 获取后端生成的所有解数据
2. **数据采样优化**：最多显示 3000 个点（从大量数据中均匀采样），避免渲染崩溃
3. **行业类型适配**：根据轻工业/重工业显示不同的指标标签
4. **交互功能**：点击帕累托解可选择对应方案
5. **动态坐标轴**：根据数据范围自动计算坐标轴范围

**关键代码**：

```typescript
// 采样函数：均匀采样数据
function sampleData<T>(data: T[], maxCount: number): T[] {
  if (data.length <= maxCount) return data
  const step = data.length / maxCount
  const result: T[] = []
  for (let i = 0; i < maxCount; i++) {
    const index = Math.floor(i * step)
    result.push(data[index])
  }
  return result
}

// 获取所有解数据
useEffect(() => {
  if (taskId) {
    setLoading(true)
    tianchouService
      .getAllSolutions(taskId)
      .then((data) => {
        setTotalCount(data.total_count || 0)
        const sampled = sampleData(data.solutions || [], MAX_POINTS)
        setAllSolutionsData(sampled)
      })
      .finally(() => setLoading(false))
  }
}, [taskId])
```

## 注意事项

### 1. Docker 容器代码同步

由于 Docker 容器内的代码不会自动同步本地修改，修改后需要手动复制文件到容器：

```bash
docker cp backend/app/models.py guanlan-sina-backend-1:/app/app/models.py
docker cp backend/app/api/routes/tianchou.py guanlan-sina-backend-1:/app/app/api/routes/tianchou.py
docker restart guanlan-sina-backend-1
```

### 2. 数据库迁移

新增字段需要运行数据库迁移：

```bash
docker exec guanlan-sina-backend-1 alembic revision --autogenerate -m "add all_solutions"
docker exec guanlan-sina-backend-1 alembic upgrade head
```

### 3. 性能优化

- 采样数量限制为 3000 个点，避免前端渲染崩溃
- 原始数据量显示在界面上，如 `3000 (采样自 20000)`
