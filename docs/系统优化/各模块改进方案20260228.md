# 天工·弈控 各模块改进方案

> 基于：`系统需要改进的地方20260301.txt` 分析 + 现有代码结构审查
> 文档日期：2026-02-28
> 优先级：🔴 高 / 🟡 中 / 🟢 低

---

## 一、核心问题总结

当前系统的三个根本性偏差：

| 问题 | 现状 | 目标 |
|------|------|------|
| 产品定位 | 像"能力售卖商店" | 工业生产智控中枢 |
| 数据状态 | 大量 Mock/静态数据 | 动态演示感（哪怕仍是模拟数据，需有节奏、有联动） |
| 业务流程可见性 | 各模块孤立，流程断裂 | 「不良发现→归因→决策→仿真验证」闭环清晰可感知 |

---

## 二、业务主流程（改进北极星）

```
目标检测发现不良
    ↓
结果回传中枢（司南·洞微）
    ↓
知识图谱归因分析（格物图谱）
    ↓
短期方案 / 长期方案生成
    ↓
多目标决策：交期 × 成本 × 品质（天筹）
    ↓
推荐方案 + 各方案预期损失展示
    ↓
仿真验证：优化前后布局/任务对比（浑天）
    ↓
执行下发（智行）
```

所有改进均需服务于这条主线的**可见性和连贯性**。

## 三、各模块改进方案

### 3.1 整体导航与业务流程可见性 🔴

**问题：** 菜单缺少业务含义，用户不知道各功能在流程中的位置，系统像能力商店。

**改进方向：**

#### A. Sidebar 增加业务流程视觉引导
- 在侧边栏顶部或折叠区增加**业务闭环迷你地图**：一张 5 步流程图（图标+箭头），点击对应步骤跳转对应页面
- 菜单项增加副标题，例如：
  - `格物图谱` → 副标题「归因分析」
  - `天筹优化` → 副标题「方案与决策」
  - `浑天仿真` → 副标题「验证对比」

**涉及文件：**
- `frontend/components/Sidebar.tsx` — 菜单结构与副标题

#### B. Dashboard 增加「决策中心」固定入口
- 在 Dashboard 首屏设置**业务状态栏**，显示：当前最新异常 / 当前推荐方案 / 当前执行状态
- 该区域作为全系统状态的"一眼摘要"

**涉及文件：**
- `frontend/pages/Dashboard.tsx`

#### C. 订阅模块边界梳理
- 明确「能力商店（Marketplace）」与「组件管理（SubscriptionValue）」的差异
- 能力商店：展示"我可以用什么能力、该能力在流程中的位置"
- 订阅复盘：展示"我订阅的能力产生了什么价值"

---

### 3.2 生产可视化（Dashboard）🔴

**问题：** 缺乏厂区-生产线动态模拟图，只有静态数据卡片。

**改进方向：**

#### A. 厂区层级可视化
- 增加**厂区/车间/产线**三层级平面图（2D或2.5D）
- 设备以图标块呈现，点击可下钻查看详情
- 层级切换：厂区全貌 → 车间 → 单条产线

#### B. 动态状态效果
- 设备状态用颜色区分：🟢运行 / 🟡待机 / 🔴故障 / ⚫离线
- 物料流动方向以流动箭头或粒子动画表示
- 产线节拍以定时刷新（非实时）的数字动效体现节奏感

#### C. 与已接入设备联动
- 已通过 KernelConnect 接入的设备，在此图中用**加粗边框/特殊图标**标注
- 点击可跳转到 OS 内核接入状态详情

#### D. 与仿真/决策结果联动
- 当天筹有推荐方案时，在产线图上**高亮「受影响产线/设备」**，鼠标悬停显示方案摘要

**涉及文件：**
- `frontend/pages/Dashboard.tsx`
- `frontend/components/LayoutVisualizer.tsx` — 扩展布局可视化能力
- `frontend/components/DeviceLayoutVisualizer.tsx`

### 3.3 智能决策（天筹 Tianchou）🔴

**问题：** 决策结果不突出，用户难以在第一时间感知推荐方案；任务配置维度过少。

**改进方向：**

#### A. 决策结果显眼展示
- 在天筹页顶部增加**决策摘要横幅**：
  ```
  推荐方案：方案B（折中方案）
  推荐理由：交期与成本平衡最优
  预期损失：总成本 ¥320万 / 工期 18天 / 预期收益 ¥580万
  ```
- 方案卡片（SolutionCard）增加**「当前最优」标签**，突出 TOPSIS 排名第一的方案

#### B. 扩展任务配置维度（TaskConfigForm）
当前配置项较少，建议扩展以下维度：

| 配置维度 | 建议字段 |
|---------|---------|
| 基础任务 | 行业类型、产线选择（已有） |
| 时间约束 | 交期（天）、换产时间、班次约束 |
| 优先级 | 订单优先级、紧急插单标识 |
| 设备约束 | 指定设备/工位、设备可用率约束 |
| 批次设置 | 批次大小、批次间依赖关系 |
| 目标权重预设 | 快速模板：「交期优先」「成本优先」「品质优先」 |

**涉及文件：**
- `frontend/pages/Tianchou/components/TaskConfigForm.tsx` — 扩展配置字段
- `frontend/pages/Tianchou/components/SolutionCard.tsx` — 增加推荐标签
- `backend/app/api/routes/tianchou.py` — 接收更多参数字段

#### C. 短期/长期方案对比列表
- 当前只有帕累托解列表，缺少**短期应急方案 vs 长期优化方案**的维度区分
- 建议在方案列表增加「方案类型」标签（短期/长期/折中）

#### D. 决策结果跨模块传播
- 天筹完成决策后，推送「当前推荐方案」到 Dashboard 状态栏
- 浑天仿真可直接加载「按天筹推荐方案推演」

### 3.4 格物图谱（KnowledgeGraph）🟡

**问题：** 页面初始化时节点全部展开，视觉过载；动态效果杂乱无节奏。

**改进方向：**

#### A. 信息分层展示
- **初始状态**：只展示「当前归因结论（根因节点）」，2~3个核心节点
- **展开操作**：点击根因节点后，以**递进动画**展开关联节点（先关联原因，再关联证据）
- **Tab 分层**：增加 Tab 切换「根因分析 / 关联异常 / 历史案例」，避免一屏堆满

#### B. 动画节奏控制
- 节点展开动画：每批次间隔 200ms，避免同时闪动
- 高亮传播：从根因节点向外辐射，有方向感
- 禁止自动循环动画（当前图谱可能有持续闪烁问题）

#### C. 与方案决策联动
- 每个根因节点旁标注「支撑方案」：点击可跳转到对应的天筹方案
- 在天筹方案卡片上增加「查看归因依据」反向入口

**涉及文件：**
- `frontend/pages/KnowledgeGraph.tsx`
- `frontend/components/KnowledgeGraphCanvas.tsx` — 动画控制
- `frontend/components/NodeDetailPanel.tsx`

### 3.5 浑天仿真（Huntian）🟡

**问题：** 缺乏优化前后的直观对比，无法体现"仿真验证"的核心价值。

**改进方向：**

#### A. 布局对比视图
- 增加**左右分屏**或**切换模式**：「原始布局 ↔ 优化布局」
- 布局差异高亮标注：设备位移用橙色箭头标出，新增/移除设备用绿色/红色边框
- 配合文字摘要：「物流方案摘要」「布局变更摘要（共移动 N 台设备）」

#### B. 任务动态甘特对比
- 增加**优化前后任务时序对比**：横轴为时间轴，两行分别展示方案A和方案B的排程
- 点击时间段高亮对应设备/产线

#### C. 产品切换场景
- 增加「产品切换」入口：选择新产品型号后，仿真展示工艺变化对产线的影响
- 增加「轻资产/重资产」选择，分别展示：
  - 轻资产：设备布局重排方案（物料搬运成本对比）
  - 重资产：AGV调度和生产排程优化方案

#### D. 与天筹联动
- 仿真页增加「加载天筹推荐方案」快捷按钮
- 仿真完成后，结果自动同步到天筹「各方案预期损失」

**涉及文件：**
- `frontend/pages/Huntian.tsx`
- `frontend/components/LayoutVisualizer.tsx` — 复用布局可视化做对比
- `frontend/components/AGVPathVisualizer.tsx`

### 3.6 场景编排（ScenarioBuilder）🟡

**问题：** 编排内容与具体产线脱节；不同编排结果未能驱动不同的仿真和优化方案。

**改进方向：**

#### A. 与产线绑定
- 编排画布中每个节点/动作支持选择「绑定产线/工位/设备」
- 在节点配置面板增加「适用产线」下拉（复用 ProductionLineSelector）
- 执行动作（机械臂、AGV）直接从已接入设备列表中选取

#### B. 编排→仿真/优化联动
- 保存编排方案后，出现选项按钮：
  - 「基于此编排生成仿真方案」→ 跳转浑天，自动加载编排参数
  - 「基于此编排生成优化任务」→ 跳转天筹，自动填充约束与目标
- 在仿真页面「选择推演方案」下拉中，列出已保存的场景编排

#### C. 增加决策输出节点
- 场景编排的节点类型中增加「决策输出」类型
- 流程运行结果中显示「本次方案」与「预期损失」摘要
- 支持跳转到天筹决策中心查看详情

**涉及文件：**
- `frontend/pages/ScenarioBuilder.tsx`

### 3.7 OS 内核接入（KernelConnect）🟡

**问题：** 扫描到的硬件设备未关联回生产线，接入过程缺少反馈。

**改进方向：**

#### A. 设备-产线映射步骤
- 在设备扫描完成后，增加「绑定产线/工位」步骤（向导式 Step 3）：
  ```
  Step 1：扫描网络设备
  Step 2：协议匹配与连接测试
  Step 3：绑定到产线/工位  ← 新增
  Step 4：完成 & 前往监控
  ```
- 每个已发现设备可选择所属车间、产线或工位
- 绑定信息写入数据库，供 Dashboard 生产可视化使用

#### B. 生产可视化反向展示
- Dashboard 产线图中，已接入设备用**绿色加粗边框**标注
- 点击设备可查看：OS内核接入状态、协议类型、最后通信时间

#### C. 接入过程反馈
- 扫描过程显示：已发现 N 台设备 / 进度百分比
- 诊断报告中明确标注「未发现设备」「连接失败设备」及建议操作
- 完成后主动引导：「前往生产可视化查看设备状态」

**涉及文件：**
- `frontend/pages/KernelConnect.tsx`
- `backend/app/api/routes/production.py` — 增加设备-产线绑定 API

### 3.8 能力商店（Marketplace）🟢

**问题：** 过于强调"售卖"，与核心业务流程脱节。

**改进方向：**

#### A. 能力卡片增加流程定位
每张能力卡片增加「在流程中的位置」标签，例如：
- 洞微·视觉检测 → `① 发现不良`
- 格物·知识图谱 → `② 归因分析`
- 天筹·运筹优化 → `③ 方案与决策`
- 浑天·仿真验证 → `④ 验证对比`

#### B. 已订阅能力的流程入口
- 已订阅能力在对应功能页面中有**明显的「由[能力名]提供」**标注
- 能力卡片上增加「前往使用」直达按钮，跳转到对应功能页

#### C. 弱化售卖气质
- 去除或弱化定价相关 UI
- 将页面标题从「能力商店」改为「能力中心」或「能力图谱」
- 增加端到端业务流示例：一张流程图展示各能力如何协同完成「异常检测→归因→推荐方案」

**涉及文件：**
- `frontend/pages/Marketplace.tsx`

## 四、技术债务与代码质量改进

### 4.1 Mock 数据治理 🔴

| 位置 | 内容 | 建议 |
|------|------|------|
| `backend/app/api/routes/anomalies.py` | `MOCK_ANOMALY_DATA` 硬编码 | 改为从数据库读取，保留 Mock 作为 `seed` 数据初始化 |
| `backend/app/api/routes/simulation.py` | `DEFAULT_SIMULATION_SCENARIOS` | 同上，迁移到数据库 `simulation_scenarios` 表 |
| `frontend/mockData.ts` | 前端 Mock 数据 | 逐步替换为真实 API 调用，或明确标注演示模式 |

### 4.2 冗余文件清理 🟢

| 文件 | 问题 | 操作 |
|------|------|------|
| `frontend/pages/Tianchou.tsx.bak` | 备份文件混入代码库 | 删除，纳入 `.gitignore` |
| `frontend/components/LandingPage - 副本.tsx` | 文件名含空格，存在引用风险 | 删除（确认主版本已是最新）|

### 4.3 算法双框架整合 🟢

- 当前 `part1_optimization.py` 用 DEAP，`part2_decision.py` 用 pymoo，两套优化框架并存
- 建议评估是否能统一到一套（优先保留与当前 API 耦合更深的 DEAP），或明确各自职责边界，避免未来维护混乱

### 4.4 AI 助手恢复规划 🟢

- `App.tsx` 中 `showGlobalAi = false` 已全局禁用 AiAssistant
- 建议在司南分析（SinanAnalysis）页面单独启用，作为归因分析的辅助工具

## 五、改进优先级排期建议

### Phase 1（演示级改进，1~2周）
> 目标：让系统看起来像一个完整的工业智控系统

1. **Sidebar** 增加业务流程迷你地图和菜单副标题
2. **Dashboard** 增加设备布局可视化（复用已有 LayoutVisualizer 组件）和设备状态色
3. **天筹** 决策摘要横幅突出显示推荐方案
4. **格物图谱** 调整初始化：只展示根因节点，点击再展开

### Phase 2（交互深度改进，2~4周）
> 目标：核心业务闭环可走通、各模块有数据联动

1. **浑天仿真** 增加优化前后布局对比视图
2. **场景编排** 增加与产线绑定和联动仿真入口
3. **KernelConnect** 增加设备-产线映射步骤
4. **天筹** 扩展 TaskConfigForm 配置维度
5. Mock 数据迁移到数据库 seed

### Phase 3（体验打磨，4~8周）
> 目标：细节体验对齐专业工业软件标准

1. 能力商店重新定位与 UI 改造
2. 格物图谱动画节奏优化
3. 浑天甘特图任务时序对比
4. 跨模块状态同步（决策结果推送、编排驱动仿真）
5. 代码冗余文件清理和算法框架整合
