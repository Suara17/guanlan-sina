# 格物图谱页面节点展示分析

## 1. 分析范围
- 页面入口：`frontend/pages/KnowledgeGraph.tsx`
- 图谱渲染：`frontend/components/KnowledgeGraphCanvas.tsx`
- 节点详情：`frontend/components/NodeDetailPanel.tsx`
- 数据适配：`frontend/services/dataAdapter.ts`、`frontend/services/neo4jService.ts`、`frontend/src/api/knowledgeGraphApi.ts`
- 后端接口：`backend/app/api/routes/knowledge_graph.py`

## 2. 图谱各节点的展示方法（当前实现）

### 2.1 节点类型与数据结构
- 节点类型固定为三类：`phenomenon | cause | solution`（`frontend/types.ts:118-125`）。
- 关系类型固定为：`leads_to | caused_by | solved_by`（`frontend/types.ts:128-134`）。
- 页面拿到数据后统一喂给 `KnowledgeGraphCanvas` 渲染（`frontend/pages/KnowledgeGraph.tsx:433-438`）。

### 2.2 节点生成逻辑（数据到图谱）
- 单异常场景：`KnowledgeGraphAdapter.transformNeo4jToFrontend()` 将 Neo4j 的异常分析结果转成前端图谱结构（`frontend/services/dataAdapter.ts:186-265`）。
- 全景场景（无 `anomalyId`）：
  - 优先拉全部异常，再由 `convertAnomaliesToGraph()` 现场拼图（`frontend/pages/KnowledgeGraph.tsx:24-127`）。
  - 节点上限 `MAX_NODES=200`，超出后直接截断（`frontend/pages/KnowledgeGraph.tsx:22,35-41,43-45`）。
- 节点/边语义：
  - 现象 -> 原因：`caused_by`
  - 原因 -> 方案：`solved_by`
  （见 `frontend/pages/KnowledgeGraph.tsx:79-87,107-117`）

### 2.3 节点视觉编码（形状/颜色/文案）
在 `KnowledgeGraphCanvas` 里由 D3 手动绘制 SVG：
- `phenomenon`：圆形 + 红色渐变（`frontend/components/KnowledgeGraphCanvas.tsx:228-235`）。
- `cause`：旋转圆角矩形（菱形感）+ 橙色渐变（`frontend/components/KnowledgeGraphCanvas.tsx:236-249`）。
- `solution`：胶囊圆角矩形 + 蓝绿渐变（`frontend/components/KnowledgeGraphCanvas.tsx:249-261`）。
- 选中态：不同类型使用不同描边色高亮（同上代码段）。
- 标签：节点中心白色粗体；长文本拆成两行 `tspan`（`frontend/components/KnowledgeGraphCanvas.tsx:264-295`）。

### 2.4 布局与分布策略
- 默认模式：力导向布局（`forceLink + forceManyBody + forceCenter + forceCollide`），并限制在矩形画布范围（`frontend/components/KnowledgeGraphCanvas.tsx:94-106,332-336`）。
- 紧凑模式（全景）：
  - 先用 Vogel（向日葵）分布做初始落点（`frontend/components/KnowledgeGraphCanvas.tsx:107-124`）。
  - 叠加 `forceRadial` 维持圆盘形态（`frontend/components/KnowledgeGraphCanvas.tsx:127-138`）。
  - tick 中再做圆形边界回拉，防止节点飞出（`frontend/components/KnowledgeGraphCanvas.tsx:315-330`）。

### 2.5 关系线和关系标签
- 边用线段 + 箭头 marker（`frontend/components/KnowledgeGraphCanvas.tsx:72-87,159-170`）。
- 边标签放在线段中点，附白色背景矩形提高可读性（`frontend/components/KnowledgeGraphCanvas.tsx:172-207,345-355`）。
- 文案映射：`caused_by -> 源于`，`solved_by -> 解决`（`frontend/components/KnowledgeGraphCanvas.tsx:195-206`）。

### 2.6 交互展示
- 节点支持点击、拖拽、hover 放大动画（`frontend/components/KnowledgeGraphCanvas.tsx:141-157,218-220,297-310`）。
- 页面接收点击后更新 `selectedNode`，并在右侧浮层展示详情（`frontend/pages/KnowledgeGraph.tsx:218-221,442-455`）。
- `NodeDetailPanel` 按节点类型切换头部配色、图标和额外信息区块（`frontend/components/NodeDetailPanel.tsx:19-50,55-127`）。

## 3. 目前用的技术

### 3.1 前端展示层
- React 19 + TypeScript（`frontend/package.json:24-25,36`）。
- D3 v7 做 SVG 力导图和交互动画（`frontend/package.json:20`，`frontend/components/KnowledgeGraphCanvas.tsx`）。
- TailwindCSS 工具类做页面和详情面板样式（`frontend/pages/KnowledgeGraph.tsx`、`frontend/components/NodeDetailPanel.tsx`）。
- `lucide-react` 图标库用于状态与类型图标（`frontend/package.json:23`）。

### 3.2 数据获取与适配
- Axios API 客户端 + 拦截器注入 token（`frontend/src/api/knowledgeGraphApi.ts:69-88`）。
- `KnowledgeGraphAdapter` 做“Neo4j优先、Mock降级”的双路数据适配（`frontend/services/dataAdapter.ts:22-50`）。
- 前端可感知数据来源并在 UI 打标“实时数据/演示数据”（`frontend/pages/KnowledgeGraph.tsx:367-377`）。

### 3.3 后端与图数据库
- FastAPI 路由提供图谱分析接口（`backend/app/api/routes/knowledge_graph.py`）。
- Neo4j + Cypher 查询异常、原因、方案及其关系（如 `CAUSED_BY`、`SOLVED_BY`，见 `backend/app/services/neo4j_service.py:215-339`）。

## 4. 当前达到的效果

### 4.1 已实现效果
- 能稳定展示“现象 -> 原因 -> 方案”三层因果链。
- 能在单异常视图与全景视图间切换，并针对全景启用紧凑圆盘布局。
- 节点形状、颜色、文案有清晰语义分层，支持选中高亮。
- 交互完整：点击、拖拽、悬停放大、右侧详情联动。
- Neo4j 不可用时自动降级到 Mock，页面可继续展示。

### 4.2 现状边界（从代码可见）
- 全景展示节点被硬限制到 200（不是动态分页/聚合）。
- 方案到原因的连接存在简化映射（按索引取模或首个原因），不是基于真实权重推理。
- 详情面板中的“时长/成功率/风险/工具”等信息部分是固定文案，不全是实时计算。
- 边标签语义有一处不一致：数据写入常见“导致”，渲染时 `caused_by` 显示“源于”。

## 5. 结论
格物图谱页面目前采用的是“React 页面状态控制 + D3 自绘力导向图 + Neo4j/Mock 双数据源适配”的实现路线。节点展示在可视语义和交互层面已经可用，尤其全景模式的圆盘铺排做了专门优化；但在超大图扩展性、关系精准建模和详情信息动态化方面还有进一步工程化空间。
